# Coding Agent Instructions (Jules)

**MISSION DIRECTIVE:** You are executing a comprehensive architecture port and refactor from Repo A (legacy React/Vite) into Repo B (`sg-condo-advisor`, an Astro + Hono + Honi Agents workspace).

Read these mandatory standards carefully. They are your immutable laws for this repository.

## 1. THE CORE DIRECTIVES & ARCHITECTURE
- **Goal:** Port legacy logic into a blazing-fast edge-native application on Cloudflare.
- **No Frontend Authentication:** The frontend must be frictionless and instantly accessible. Remove login/session overhead from the UI. (Backend auth capabilities for external MCP clients can remain toggleable via env vars, but the web UI is public).
- **Frameworks:** Astro (Static HTML shell), React Islands (`client:load` for interactivity), Hono (`@hono/zod-openapi`), Drizzle ORM.
- **AI Framework:** You MUST use the `honidev` package for all agents. The legacy Cloudflare Agents SDK is forbidden.

## 2. SECRETS, ENV MANAGEMENT & TYPE SAFETY
- **Secrets Store Bindings:** You must use the Cloudflare Secrets Store bindings defined in `wrangler.jsonc`. To access secrets in code, ALWAYS use the async `.get()` method: `await env.{SECRET_BINDING_NAME}.get()`.
- **Wrangler Types:** ALWAYS run `npx wrangler types` after ANY change to `wrangler.jsonc` (bindings, vars, etc.).
- **Global Env:** The generated `worker-configuration.d.ts` is the SINGLE SOURCE OF TRUTH. Ensure it is installed correctly and included in `tsconfig.json`. NEVER redefine or manually type the `Env` interface.

## 3. STRICT IMPORT PATHS
- **Global Path Aliases Only:** EVERY module import MUST use the TypeScript path aliases defined in `tsconfig.json`.
- ✅ **CORRECT:** `import { db } from '@/lib/db'` or `import { properties } from '@/schema/properties'`
- ❌ **FORBIDDEN:** `import { db } from '../../../../lib/db'` 
- *Never* use relative imports for traversing up directories.

## 4. DATABASE & DRIZZLE ORM
- **Folder Structure:** The schema must be completely modularized. Each table must be represented as a separate TypeScript file inside a `schema/` directory (e.g., `schema/users.ts`, `schema/healthChecks.ts`).
- **Tooling:** You must ONLY use Drizzle Schema and Drizzle ORM.
- **Live Sampling:** The DB points to an existing database with populated data. **You must sample the DB** using the API key and Account key set in your environment to ensure your newly modularized schema perfectly matches the existing database configuration.
- **Mandatory Scripts:** Ensure `package.json` strictly uses these scripts for database operations and deployment:
  - `"db:generate": "drizzle-kit generate"`
  - `"migrate:remote": "npm run db:generate && wrangler d1 migrations apply DB --remote"`
  - `"deploy": "npm run build && npm run migrate:remote && npx wrangler deploy"`

## 5. AI, HONI AGENTS & TOOLS
- **AI Gateway Integration:** For the Honi agents, you MUST route all AI model calls through the AI Gateway using the client located at `src/lib/ai/utils/ai-gateway-client.ts`. Do not call OpenAI, Anthropic, or Workers AI directly without passing through this gateway client.
- **Browser Rendering Tool:** The Honi agent must be equipped with a browser rendering tool to research real-time condominium information. You must implement this tool using the Browser Rendering API wrapper located at `src/lib/cloudflare/browser.ts`.
- **MCP Endpoints:** The Honi agent must expose standard MCP endpoints (`/mcp` and `/sse`) with tools like `calculate`, `healthCheck`, `queryDatabase` (STRICTLY read-only), and `researchCondo` (using the browser render tool).

## 6. FRONTEND RETROFITTING (STITCH MOCKUPS -> ASTRO/SHADCN)
- **Mockup Ingestion:** You will pull down frontend HTML mockups generated by UX agents (like Stitch).
- **Retrofit Protocol:** You are REQUIRED to retrofit those raw HTML mockups into the Astro + React Islands architecture.
- **Strict shadcn/ui Usage:** You must manually translate raw HTML elements from the mockups into default dark theme `shadcn/ui` React components. 
  - *Example:* If the Stitch HTML uses `<button class="bg-blue-500...">Submit</button>`, you MUST replace it with `import { Button } from "@/components/ui/button"` and use `<Button>Submit</Button>`.
- **Dark Theme:** The global layout must enforce `<html class="dark">`.

## 7. REQUIRED PAGES & ISLANDS
When building the frontend, you must implement the following layout and pages:
1. **Global Sidebar Navigation:** Persistent left sidebar linking to Home, Dashboard, Chat, Admin, MCP Tester, and Swagger UI.
2. **`/chat` (Agent Chat):** Publicly accessible React Island using `assistant-ui` `<Thread>`, wrapped in a shadcn `<Card>`, connecting to the Honi Agent WebSocket.
3. **`/mcp-tester` (NEW):** A React Island specifically for developers to visually list, configure JSON payloads, and execute MCP backend tools directly from the browser.
4. **`/dashboard`:** An observability React Island displaying D1 logs, system health checks (green/red border cards), and MarketScanWorkflow traces.
5. **`/admin`:** A settings panel to dynamically update the AI Gateway/model configurations (saved to KV).
